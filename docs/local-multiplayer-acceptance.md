# 上号本地多人验收清单

这份清单用于发布前验收。自动测试不能代替真实公网、真实声卡和真实 Windows 设备测试。

## 1. 自动预检

在仓库根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/verify-local-acceptance.ps1 -Package
```

通过标准：类型检查、桌面端回归、三人信令音频路由、生产构建、Windows 打包和包内资源校验全部为绿色。

## 2. 设备准备

- 最少两台独立 Windows 电脑；三人测试建议第三台电脑、Windows Sandbox 或独立虚拟机。
- 每台设备安装同一个本地测试包，使用不同昵称和不同角色。
- 每台设备填写同一个固定频道服务器地址。
- 准备一副耳机，避免扬声器回授被误判为降噪故障。
- 测试前记录 Windows 缩放比例、网络类型、是否开启 Clash/Mihomo/TUN。

## 3. 双机基础链路

1. A、B 依次进入频道，确认两个角色占用不同工位且不会重叠。
2. 双方轮流说话 2 分钟，确认说话状态、真实音量波形、静音红色状态一致。
3. 双方同时说话 30 秒，确认声音不会互相截断、爆音或明显抽搐。
4. 切换麦克风和扬声器设备，确认远端声音不消失，延迟标签不因换座短暂清空。
5. B 退出后重新进入，确认 A 不崩溃，B 得到唯一座位并重新听到语音。

## 4. 三人和五人房间

1. A、B、C 同时在线并轮流说话，每人至少 1 分钟。
2. 三人同时说话 30 秒，确认共享 AudioMixer 不丢失任意成员。
3. 有条件时加入 D、E，确认五个工位唯一、没有角色叠座。
4. 连续通话至少 60 分钟，记录 CPU、GPU、内存峰值和是否出现音频卡顿。

## 5. 断线和代理恢复

1. B 关闭 Wi-Fi 15 秒再打开，确认界面显示重连，恢复后原成员身份和座位不重复。
2. B 从 Wi-Fi 切到手机热点，确认无需重启软件即可恢复。
3. 开启和关闭 Clash/Mihomo/TUN 各一次，确认固定国内服务器仍能恢复连接。
4. 网络恶化时界面只显示“流畅 / 一般 / 不稳定”，语音优先于屏幕画质降级。

## 6. 屏幕分享

1. A 选择 720p 分享，B、C 同时观看，并保持三人语音 5 分钟。
2. A 选择 1080p 分享，观察弱网时是否自动降帧、降码率且不挤占语音。
3. 分享窗口时勾选系统音频，确认 B、C 能听到系统声音。
4. B 打开独立观看窗口，确认它可在软件外移动、缩放并直接播放媒体流。
5. 尝试选择上号自身窗口，确认来源列表不提供该窗口；分享整个显示器时不出现无限镜像。
6. A 停止分享，确认所有观看端立即恢复场景且没有残留黑窗。

## 7. 视觉和 DPI

- 分别在 100%、125%、150% Windows 缩放下检查首页、房间、设置、悬浮窗和分享窗口。
- 缩放窗口到最小尺寸，确认五个工位、离开区、聊天输入和底栏操作不被截断。
- 检查冰川蓝环境光、顶底栏玻璃、偏实体聊天区和主场景层级是否清楚。
- 检查角色入场、换座、离开、欢迎转头、聊天气泡、敲一下和屏幕分享动画是否连续。

## 8. 记录结果

每轮记录：安装包文件名、三台设备系统版本、DPI、网络、代理状态、开始/结束时间、失败步骤和诊断包路径。任何崩溃、重复座位、持续无声或屏幕黑屏都视为阻断发布。

# 上号房主直连说明

房主直连的意思是：你的电脑启动上号房间，朋友从外网直接连到你的电脑。

这里真正关键的不是 `https`，而是你的电脑能不能被公网访问到。

## 能直接连上的三种情况

1. 你和朋友都有可用公网 IPv6。
   - 软件会生成类似 `ws://[2409:...]:43821/?roomId=...` 的地址。
   - Windows 防火墙需要允许上号入站。
   - 路由器如果有 IPv6 防火墙，也要允许 TCP `43821` 入站。

2. 你有公网 IPv4，并且路由器做了端口映射。
   - 路由器 WAN IP 要和公网 IP 检测结果一致。
   - 把 TCP `43821` 映射到你电脑的局域网 IP，比如 `192.168.1.2:43821`。
   - Windows 防火墙允许上号入站。

3. 你有公网 IPv4，并且 UPnP / NAT-PMP 自动映射成功。
   - 软件会尝试自动映射 TCP `43821`。
   - 如果路由器关闭了 UPnP，自动映射会失败，需要手动映射。

## 不能靠本地电脑直连的情况

如果你的宽带在运营商 CGNAT 后面，就算软件写得再好，朋友也不能从公网直接打到你的电脑。

常见表现：

- 路由器 WAN IP 是 `10.x`、`100.64.x`、`172.16-31.x`、`192.168.x`。
- 公网 IP 检测出来的地址和路由器 WAN IP 不一致。
- 路由器里做了端口映射，但外网仍然连不上。

这种情况下只能改用：

- 云服务器中继。
- Tailscale / WireGuard 这类组网。
- 运营商改公网 IP。

## `ws`、`wss`、`https` 到底是什么关系

- `ws://` 是普通 WebSocket。
- `wss://` 是加密 WebSocket，类似 `https`。
- 上号桌面端用的是 Electron/Node WebSocket，不是普通浏览器网页，所以 `ws://公网IP:43821` 本身可以用。
- `wss://` 更适合云服务器、域名和 443 端口，但它不能自动解决 NAT、端口映射、CGNAT 问题。

一句话：`wss` 更正规，但“朋友能不能连到你的电脑”首先取决于公网入口是否打通。

## 建议验证顺序

1. 开房后先看软件生成的地址。
2. 如果地址是 `127.0.0.1`，一定不能发给朋友。
3. 如果地址是 `192.168.x.x`，只适合同一局域网。
4. 如果地址是 `[2409:...]` 这类 IPv6，可以让有 IPv6 的朋友直接试。
5. 如果地址是公网 IPv4，需要确认路由器 TCP `43821` 已映射到你的电脑。
6. 如果以上都失败，基本就是公网入口不具备，应该用云服务器入口。

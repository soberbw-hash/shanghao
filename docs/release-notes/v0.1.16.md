# 上号 v0.1.16

这版是一次直接针对“开房后地址一直卡在检测中”的热修。

## 这次修了什么

- `direct_host` 现在会优先给出局域网可用地址  
  只要本机有可用的私网 IPv4，房间启动后就会先生成可分享地址，不再一直卡在“等待地址准备完成”。

- 房间启动和公网探测彻底拆开  
  现在会明确区分：
  - 房间本地已启动
  - 局域网地址已就绪
  - 公网直连能力仍在后台检测

- 页面地址展示加了一层兜底  
  即使主进程还没把 `signalingUrl` 字段完整写回，只要已经有 `hostAddress + port + roomId`，界面也会立刻拼出邀请地址并允许复制。

- 局域网回退不再误报成“公网直连已就绪”  
  如果当前只是局域网地址可用，页面会明确提示这是同一网络下可用的地址。

## 建议怎么测

1. 房主升级到 `v0.1.16`
2. 选择“房主直连”后点击“开启房间”
3. 看房间地址卡片是否立刻出现一个可复制地址
4. 让同一局域网下的好友先试一次加入
5. 如果你们不是同一局域网，再改用 `Tailscale`

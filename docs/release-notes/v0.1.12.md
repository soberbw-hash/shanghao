# 上号 v0.1.12

这是一个高优先级热修版，专门修复“房主分享出去的地址竟然是 `127.0.0.1`，导致朋友根本加不进来”的问题。

## 这次修了什么

- 修复房主开房后把本地回环地址 `ws://127.0.0.1:...` 当成邀请地址分享出去的问题
- 房主自己仍然可以走本地回环地址进房，减少 NAT hairpin 影响
- 对外分享、复制、展示的地址现在只会走真正可分享的邀请地址
- 补了一条回归测试，防止 `localhost` 再次泄漏到邀请链路

## 这次问题的根因

房主本机开房时，程序会为了让房主自己更稳定进房，优先走本地 `127.0.0.1` 连接 signaling。

这个策略本身没错，错在后面把这条“仅房主自己可用”的本地地址写回了房间状态，导致：

- 复制地址时复制成了 `127.0.0.1`
- 朋友收到后当然连不上
- Tailscale 模式下也会被这个问题误伤

## 现在正确行为

- 房主自己连接房间：可以继续走本地回环地址
- 对外邀请地址：只显示真正给朋友用的地址
- 如果房主直连公网探测还没完成，不会再提前给出错误的 localhost 邀请地址

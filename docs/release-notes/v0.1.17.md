# 上号 v0.1.17

这版重点修连接，不改大方向。

## 修了什么

- 加入房间现在会按邀请链接里的候选地址依次重试，不再只试一个地址就失败。
- Tailscale 邀请链接会同时带上 MagicDNS / 100.x 等备用地址，朋友端会自动尝试可用的那个。
- WebRTC 增加默认 STUN，房主直连在常见 NAT 环境下更容易打通。
- 新增“信令音频兜底”：如果 WebRTC 音频被 NAT / 代理 / TUN 卡住，但双方已经进到同一个房间，会临时通过房间信令转发小队语音。
- 房间信令服务器现在会转发兜底音频包，并用音频包刷新心跳，避免通话中被误判掉线。

## 使用建议

- 最稳：固定好友都加入同一个 Tailscale，再用 Tailscale 模式开房。
- 有公网 IP / 端口映射：可以用房主直连。
- 如果你有腾讯云服务器：下一步可以把它作为云中继 / TURN 服务器，给所有网络环境兜底。

## 验证

- 已通过 TypeScript 类型检查。
- 已通过桌面端烟测，包含房间成员同步、地址候选、兜底音频转发测试。

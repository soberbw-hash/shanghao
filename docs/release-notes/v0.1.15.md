# 上号 v0.1.15

这版是一次直接对着“能不能连上”和“桌面图标太小”收的热修。

## 这次修了什么

- `direct_host` 不再等公网探测完成才给地址  
  现在房间本地一启动，就会先给出手填地址或局域网候选地址；公网直连能力继续在后台检测。

- 房主不会再一直卡在“正在开启...”  
  “房间已启动”和“公网地址已验证”现在是两段状态，不会再混成一个 pending。

- 当公网不通时，仍会保留当前可用候选地址  
  如果公网探测失败，但局域网候选地址还在，界面会明确提示这是局域网候选地址，并建议切到 Tailscale 或云中继。

- 桌面快捷方式图标重新放大并刷新链路  
  这次重新导出了更大主体占比的图标，并更新了安装器快捷方式图标资源路径，减少 Windows 复用旧缓存的概率。

## 适合怎么测

1. 房主点“开启房间”
2. 看看是不是立刻能看到一个地址候选，而不是一直空着
3. 同一局域网先试一次直连
4. 如果公网不通，再切到 Tailscale 继续测
